# Lynx Pretext

[Pretext](https://github.com/chenglou/pretext) 是一个纯 JavaScript 实现的文本测量和布局库，专为浏览器设计。它提供了精确的文本分段、换行算法和布局计算，特别适合需要精细控制文本排版的场景——比如杂志编辑器、代码编辑器、或复杂的排版系统。

它的核心架构采用两阶段设计：

```
┌─────────────────┐      ┌─────────────────┐
│  Prepare Phase  │ ──▶ │  Layout Phase   │
│  (文本出现时)   │      │  (容器变化时)    │
└─────────────────┘      └─────────────────┘
        │                        │
        ▼                        ▼
  - 文本分析/分段              纯算术计算
  - 测量每段文本               遍历缓存宽度
  - 缓存结果                   计算行数和高度
```

这种设计让文本布局可以高效地响应容器尺寸变化，而无需重新测量文本。

### 适配

Lynx 是一个跨平台移动应用框架，它的运行环境和浏览器有本质区别：

| 特性 | 浏览器 Pretext | Lynx Pretext |
|------|---------------|--------------|
| **测量 API** | Canvas `measureText()` | `lynx.getTextInfo()` |
| **运行环境** | 浏览器主线程/Worker | Lynx 主线程 (MTS) |
| **字体配置** | `ctx.font = font` | 通过 `fontSize`/`fontFamily` 参数 |
| **Emoji 修正** | 自动检测 Canvas/DOM 差异 | MVP 版本不支持 |

更重要的是，Lynx 采用独特的**双线程架构**：

- **Background Thread (BTS)**: React 运行环境，负责声明式渲染
- **Main Thread (MTS)**: UI 渲染、触摸事件、动画，零延迟响应

这意味着文本布局代码不仅要适配新的测量 API，还要考虑在哪个线程执行、如何跨线程通信。

### 目标：最大化代码复用

我们希望尽可能复用 Pretext 的核心算法，而不是重新发明轮子。最终的代码复用率达到了 **85-90%**：

| 模块 | 原始行数 | 移植后行数 | 复用率 |
|------|---------|-----------|--------|
| `analysis.ts` | 1,008 | ~1,016 | ~95% |
| `line-break.ts` | ~1,056 | ~1,056 | ~98% |
| `layout.ts` | 718 | 621 | ~85% |
| `measurement.ts` | 232 | 149 | ~60% |

差异主要集中在测量层（从 Canvas API 迁移到 Lynx API），而核心的分析和换行算法几乎原封不动。



### 架构模式：MTS vs BTS

**核心问题**：代码应该放在哪个线程？

```
┌─────────────────────────────────────────────────────────────┐
│                    Main Thread (MTS)                        │
│  - UI 渲染、触摸事件、动画                                    │
│  - 零延迟响应触摸                                            │
│  - 可以直接操作 DOM (通过 useMainThreadRef<Element>)          │
│  - 不能访问 React state/hooks                               │
└─────────────────────────────────────────────────────────────┘
                            ↕
              runOnMainThread / runOnBackground (异步)
                            ↕
┌─────────────────────────────────────────────────────────────┐
│                  Background Thread (BTS)                    │
│  - React 运行环境                                            │
│  - useState, useCallback, useMemo 等 hooks                  │
│  - JSX 声明式渲染                                            │
│  - 文本布局计算 (pretext)                                    │
└─────────────────────────────────────────────────────────────┘
```

**三种模式对比**：

| 方面 | 纯 BTS | 纯 MTS | 混合模式 |
|------|--------|--------|----------|
| 动画位置 | BTS | MTS | MTS |
| 文本布局 | BTS | MTS (shared) | BTS |
| 触摸处理 | BTS (有延迟) | MTS (零延迟) | MTS (零延迟) |
| 跨线程同步 | 无 | 无 | 每帧 `runOnBackground` |
| 同步延迟 | - | - | 1-2 帧 |
| 代码复杂度 | 低 | 高 | 中 |
| 性能 | 依赖 reconciliation | 最高 | 高 |

**选择建议**：
1. 需要零延迟触摸响应 → 纯 MTS
2. 简单动画，触摸不重要 → 纯 BTS
3. 复杂文本布局 + 需要触摸 → 混合模式或纯 MTS
