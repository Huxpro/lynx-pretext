# Ralph Progress Log

## Codebase Patterns
- lynx.getTextInfo(text, { fontSize: "16px", fontFamily?: "Inter" }) returns { width: number, content?: string[] } — synchronous on main thread
- Module-level font context (currentFontSizeStr, currentFontFamily) must be set before calling getSegmentMetrics
- Root tsconfig.json targets ES2020 with lib ES2020 + ES2022.Intl — supports Set, Map, codePointAt, Array.from, Unicode regex flag, Intl.Segmenter types
- Rspeedy project uses `lynx.config.ts` (not `rspeedy.config.ts`) for build configuration
- Entry point is `src/index.tsx`, demo pages live under `pages/`
- `src/tsconfig.json` includes `../pages/**/*.tsx` so pages are typechecked with src
- `@formatjs/intl-segmenter` v12.x is the current major (v11 is no longer available)
- `tsc --build` is the typecheck command (project references mode)
- `import '@formatjs/intl-segmenter/polyfill-force.js'` patches Intl.Segmenter globally; use `ES2022.Intl` in lib for TS types
- Rspeedy build output goes to `dist/main.lynx.bundle`
- lynx.getTextInfo supports maxWidth as string ("300px") and returns content?: string[] with line-broken text
- TextInfo type: { fontSize: string, fontFamily?: string, maxWidth?: string, maxLine?: number }
